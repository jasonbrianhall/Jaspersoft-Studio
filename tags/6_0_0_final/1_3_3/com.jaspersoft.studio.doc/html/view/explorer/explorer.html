Project Explorer View

Jaspersoft Studio use eclipse way to organize all the resources, this means users will create projects that contain files, images, styles, libraries and other files needed for reports development. Users will use Project Explorer View in order to do that.

This view show projects file structure similar to file managers. Users should use context menus or drag and drop to work with files.

One of the current tasks is to open a report, to do this just double clicking a .jrxml or .jasper file, or selecting Open With context menu, there are two modes JasperReports Designer will open report in design mode, or JasperReports Preview will open only preview from which it's possible to run reports. It's also possible to open .jrprint files to just see a generated report, .jrtx files to edit styles, DataAdapter files to create DataAdapters, or just preview the report from Preview item in context menu.

Another important task is to add or remove java libraries jars to or from project CLASSPATH. For this open context menu on the desired jar and look at "Build Path" item, sub items are "Add To Build Path" or "Remove From Build" path menus, accordingly. Please look into eclipse documentation how to configure CLASSPATH in eclipse, use source folders and Project Explorer View for more information.

Existing eclipse users may already have some projects in the workspace, for this users it's important to know that they have to activate JasperReports Nature on the project with .jrxml files, to do this jus select the project, open context menu and look at JasperReports → Toggle JasperReports Nature menu item.

Plenty of configuration options are available for this view, to open configuration dialog click on the arrow in top right corner of the view.
